# Design System

The visual language for Scout: tokens, typography, and component patterns.

Generated by `astro-builder`. Update by re-running `/astro-builder:init` or editing
`src/styles/global.css` and this file in sync.

> **Source of truth:** `src/styles/global.css` contains the actual CSS custom properties. This
> file is the human-readable index. When tokens change, update both.
>
> **Provisional graft:** the current tokens are seeded from the locked design system
> (`design/tokens.css` + `design/DESIGN_DIRECTION.md`). A faithful graft + self-hosted Charter
> webfont + verdict-plate/serif primitives land in a follow-up (**F0b**).
> `src/styles/tokens.reference.css` is a non-imported copy of `design/tokens.css` kept for that step.
>
> **Authoring rule:** every color, font family, font size, spacing value, border radius, and box
> shadow used in component CSS must reference a token namespace below. See `css-conventions`.

## Register

**Default register**: depends on section (this site has both).
**Per-section split**:
- `brand` — the marketing landing (`/`, `HomePageView`). The page *is* the product.
- `product` — the docs reader (`/docs`, `DocsIndexPageView` + `DocPageView`). The user is *in a task*.

The register sets the defaults for type, color, motion, and layout judgment, and scopes the
`/astro-builder:audit` anti-slop step. Tag judgment per section when auditing.

## Product & brand context

- **Audience**: developers and AI coding agents adopting Scout for browser QA.
- **Primary use case**: evaluate Scout, then run it in CI as the gate for autonomous fix-to-PR loops.
- **Brand voice (three physical-object words)**: calibrated, optical, measured.
- **Named aesthetic reference**: an optical/scientific instrument's panel — cool observatory paper,
  lens-blue focus, brass calibration marks, editorial serif (the "Bring it into focus" direction).
- **Anti-references (avoid looking like)**: the near-black + single-hot-accent generic dev-tool;
  warm oat/cream grounds; night-violet; gradient-clipped headlines; mono-as-identity typeface.

## Visual style

**Aesthetic**: editorial-scientific / optical instrument.
**Description**: light-first cool observatory paper (`#F4F6F8`) with a designed night "observation
mode", a single lens-blue focus signal, brass as a calibration-only second accent, and a verdict
ramp (verified / partial / failed / blocked) as the recurring signature. Type encodes meaning:
serif for display *and* docs prose, humanist sans for UI, mono for machine output only.

foreground in both modes (never reuse `--color-text` on a plate).

## Component patterns

These compose tokens — they don't introduce new raw values.

### Cards

```css
.card {
  background: var(--color-surface);
  border: 1px solid var(--color-border);
  border-radius: var(--radius-md);
  box-shadow: var(--shadow-sm);
  padding: var(--space-lg);
}
```

### Buttons

| Variant | Background | Text | Border |
|---------|-----------|------|--------|
| Primary | `--color-primary` | `--color-on-failed` (light ink) | same |
| Secondary | transparent | `--color-primary` | `--color-primary` |
| Ghost | transparent | `--color-text` | `--color-border` |

### Verdict plates (signature — F0b will formalize)

Solid plate `background: var(--color-verified)` (etc.) **must** pair with its dedicated
`color: var(--color-on-verified)`. Vermilion (`--color-failed`) is failed-only and never a general
accent; lens-blue (`--color-primary`) is the only focus signal.

## Accessibility baseline

Enforced in `global.css` under `@layer base`:

- All interactive elements have `:focus-visible` styles (2px outline in `--color-focus`).
- Color contrast meets WCAG AA (4.5:1 for body text). Use `--color-brass-deep` for small brass text.
- `prefers-reduced-motion` disables non-essential transitions and animations.
- Skip-navigation link present in `BaseLayout.astro`.

## CSS approach

Custom CSS (the plugin default). No Tailwind, no utility-class library, no CSS-in-JS, no preprocessor.

All global tokens and resets live in `src/styles/global.css`, organized as:

```css
@layer reset, tokens, base, utilities;
```

Component styles live in Astro scoped `<style>` blocks (one per `.astro` file). See the
`css-conventions` skill for the full discipline.

### Design anti-patterns (from `design/DESIGN_DIRECTION.md` — load-bearing)

1. No verdict color as a general UI accent; vermilion is failed-only; lens-blue is the only focus signal.
2. Never run two hot layers at once — one orchestrated kinetic moment.
3. Mono is never the identity typeface — machine output only.
4. Do not warm or darken the cool paper ground; no oat/cream, no night-violet.
5. Do not number sections decoratively (the 01–04 loop is a true cycle; docs groups are not numbered).
6. No scattered motion — the machine-generated tell.
7. Solid plates keep their `--color-on-*` foreground; never reuse `--color-text` on a plate.
8. No fourth typeface, no gradient-clipped headlines.

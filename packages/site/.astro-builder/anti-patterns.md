# Anti-Patterns

What NOT to do in Scout. Every item here has caused real problems — either in this project or in
similar Astro projects.

Generated by `astro-builder`. See also `docs/astro-patterns.md` for the canonical Astro 6 list.

---

## Astro 6 anti-patterns (always forbidden)

| Anti-pattern | Correct alternative |
|---|---|
| `src/content/config.ts` | `src/content.config.ts` |
| `astro.config.mjs` | `astro.config.ts` |
| Separate collection per language | Single collection with locale subfolders (`docs/en/`) |
| Fixed `category` enum | `tags: string[]` (note: `group` IS a deliberate fixed enum for sidebar IA) |
| UI strings hardcoded in `.astro` | `tl('key')` from `src/i18n/en.json` |
| Logic or data fetching in page files | Move to a page-view (exception: `getStaticPaths` plumbing) |
| `lang` or `tl` passed as props | Each component calls `createTranslator(Astro.currentLocale)` |
| Parsing `Astro.url` for locale | `Astro.currentLocale` (fall back to `"en"`) |
| `redirectToDefaultLocale: true` | `false` (no root redirect needed — default locale is unprefixed) |
| ESLint or Prettier | Biome |
| Custom sitemap code | `@astrojs/sitemap` |
| Relative imports across `src/` | Path aliases (`@lib/`, `@components/`, etc.) |
| `// @ts-ignore` in new code | Fix the actual type error |

## Project-specific anti-patterns

These OVERRIDE or extend the canonical list for this monorepo and this site:

| Anti-pattern | Correct alternative |
|---|---|
| **`pnpm` / `yarn` / `pnpm-lock.yaml` / `pnpm-workspace.yaml`** | **`npm` workspaces** — one root `package-lock.json`, `npm ci` in CI. The plugin default of pnpm is overridden here. |
| Installing deps inside `packages/site/` ad hoc | `npm install <pkg> -w @pcamarajr/scout-site` from the repo root |
| **`prefixDefaultLocale: true`** (the plugin default) | **`prefixDefaultLocale: false`** — v1 ships English only; the default locale is unprefixed (`/`, `/docs/`). Re-enabling the prefix is a real i18n change, made deliberately when a second locale lands. |
| Pages under `src/pages/en/` | Pages at the site root (`src/pages/index.astro`, `src/pages/docs/...`) |
| Hardcoding the origin (`https://scout.pcamarajr.dev`) anywhere but `astro.config.ts` | Derive absolute URLs from `Astro.site` in `BaseLayout` |
| Touching the repo-root `CLAUDE.md`, root `package.json` scripts, `commitlint.config.js`, `.husky/`, or `packages/cli/` | Keep all site work under `packages/site/` |
| Adding an SSR adapter | Keep `output: "static"` — a Vercel static build needs no adapter; the future verdict-board island is a client-side fetch (static-compatible) |
| Importing `src/styles/tokens.reference.css` | It is a non-imported reference copy of `design/tokens.css` for the F0b graft step |
| Selling determinism / "0 LLM on replay" as the moat | Pitch the real moat combination (see `style-guide.md` honesty guardrails) |
| Verdict color as a general accent; mono as the identity typeface; warm/dark ground | See the design anti-patterns in `design-system.md` |

## Deferred (not anti-patterns — intentional omissions in v1)

- **RSS feed + `<link rel="alternate" type="application/rss+xml">`** — deferred with the dated
  `blog` collection to phase 2. There is no dated collection to feed yet.
- **hreflang alternates** — return when a second locale is added.
- **Self-hosted Charter webfont + verdict-plate/serif primitives** — the F0b faithful-graft step.

## When you encounter an anti-pattern

1. Do not work around it — fix the root cause.
2. Run the quality gates after fixing to confirm no regressions.
3. If it is in code you did not write and fixing it is out of scope, note it and create a follow-up.

Never use `--no-verify`. Never suppress TypeScript errors with `any` or `@ts-ignore` unless the
existing codebase already does so in that specific context.

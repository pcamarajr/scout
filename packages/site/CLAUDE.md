## Project Overview

Self-healing browser QA — describe a test in plain English; Scout verifies it in a real browser, records a deterministic script, and replays it for free.

**Site name**: Scout
**Target audience**: developers and AI coding agents adopting Scout for browser QA
**Deployed at**: https://scout.pcamarajr.dev
**Deployment target**: Vercel (static)

## Monorepo context

This site is one workspace in an **npm-workspaces** monorepo (`@pcamarajr/scout-monorepo`).
The CLI lives in `packages/cli` (`@pcamarajr/scout`); this is `packages/site`
(`@pcamarajr/scout-site`, private). Everything below is scoped to `packages/site/`.

- **Package manager: npm**, not pnpm. There is one root `package-lock.json` and CI runs `npm ci`.
  Never run pnpm; never create `pnpm-lock.yaml` or `pnpm-workspace.yaml`.
- Add deps from the **repo root**: `npm install <pkg> -w @pcamarajr/scout-site`.
- Do not touch the repo-root `CLAUDE.md`, root `package.json` scripts, `commitlint.config.js`,
  `.husky/`, or anything under `packages/cli/`.

## Development Commands

Run from the repo root (`-w` targets this workspace):

```bash
npm install                              # Install all workspace deps (npm only)
npm run dev -w @pcamarajr/scout-site     # Start dev server
npm run build -w @pcamarajr/scout-site   # Production build
npm run preview -w @pcamarajr/scout-site # Preview production build
```

**Quality gates** (every commit must pass):

```bash
npm run build -w @pcamarajr/scout-site      # Zero build errors
npm run typecheck -w @pcamarajr/scout-site  # astro check + tsc --noEmit, zero errors
npm run lint -w @pcamarajr/scout-site       # biome check, zero lint/format errors
npm run format -w @pcamarajr/scout-site     # biome check --write — auto-fix before committing
```

Biome is scoped to `packages/site` (config: `packages/site/biome.json`).

## Languages

- **Default locale**: en
- **All locales**: en (English only in v1)
- **URL pattern**: NO prefix on the default locale. Pages live at the site root
  (`/`, `/docs/`, `/docs/<slug>/`), **not** under `/en/`. i18n stays latent — adding a

## Architecture

### Page-views pattern (mandatory)

Three layers — never collapse them:

1. **Pages** (`src/pages/`) — Thin wrappers (≤5 lines). Import a PageView, nothing else.
   The one exception is a page that needs `getStaticPaths` (data plumbing for dynamic routes,
   e.g. `src/pages/docs/[...slug].astro`) — keep it to the path map + prop pass-through.
2. **Page-Views** (`src/page-views/`) — All markup, data fetching, and translations.
3. **Layouts** (`src/layouts/`) — Page shell (html, head, body, header, footer).

### i18n

- All UI strings in `src/i18n/en.json` — flat dotted keys (`"notFound.title"`). The default
  locale file defines the `MessageKey` type in `src/lib/i18n.ts`, so a typo in a `tl()` call is
  a compile error.
- Use `createTranslator(Astro.currentLocale)` in every component — never pass `lang` or `tl`
  as props.
- Detect locale via `Astro.currentLocale` — never parse URLs. With `prefixDefaultLocale: false`
  it may be `undefined` on default-locale routes; fall back to `"en"`.
- `routing: { prefixDefaultLocale: false, redirectToDefaultLocale: false }`.

### Content collections

- Config at `src/content.config.ts` (NOT `src/content/config.ts`).
- `glob()` loaders, Zod schemas.
- Single collection per type with locale subfolders: `docs/en/`.
- `translationKey` links content across locales.
- Flexible `tags: string[]` — no fixed category enums.
- `draft: true` keeps an entry out of listings, feeds, and sitemaps.

### Utility layer (`src/lib/`)

- `content.ts` — all content collection queries (page-views never call `getCollection()` directly).
- `urls.ts` — all URL builders (components never concatenate locale prefixes by hand). Owns the
  "default locale is unprefixed" rule.
- `format.ts` — locale-aware date/number formatting via `Intl` (never a date library).
- `i18n.ts` — `createTranslator()` factory + the `MessageKey` type.

### Path aliases (always use these)

```
@/*            → src/*
@components/*  → src/components/*
@layouts/*     → src/layouts/*
@styles/*      → src/styles/*
@lib/*         → src/lib/*
@page-views/*  → src/page-views/*
@i18n/*        → src/i18n/*
```

### CSS conventions

All CSS work follows the **`css-conventions` skill** from the `astro-builder` plugin. Key rules:

- **Tokens only:** every color, font, font size, spacing value, radius, and shadow comes from
  CSS custom properties in `src/styles/global.css`. No raw hex/rgb/px values in component CSS.
- **Single global stylesheet:** `src/styles/global.css` with `@layer reset, tokens, base,
  utilities`. No other global CSS files. No CSS Modules, no `<style is:global>`.
  (`src/styles/tokens.reference.css` is a non-imported reference copy of `design/tokens.css`
  for the F0b faithful-graft step — never import it.)
- **Component CSS** lives in Astro scoped `<style>` blocks only.
- **No Tailwind, no utility-class libraries, no CSS-in-JS, no preprocessors.**
- **Interactivity ladder:** before adding `client:*`, walk CSS → vanilla `<script>` → island.

See `.astro-builder/design-system.md` for the token index.
